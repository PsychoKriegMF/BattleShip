1. Постановка задачи.
Разработать игру "Морской бой".

2. Поля 10х10, одно свое, второе соперника.
В своём поле устанавливаются корабли: 4 клетки - 1 шт, 3 клетки - 2 шт, 2 клетки - 3 шт, 1 клетка - 4 шт.
Корабли могут распологаться вертикально и горизонтально. Расстояние между кораблями 1 клетка минимум. Предусмотреть возможность расстановки компьютером.
Ходы делаются по очереди. На поле противника указывается точка выстрела.
Варианты ответа: "Мимо", "Ранил", "Убил". При удачном выстреле ход не передается. При ответе "Убил" окружаем уничтоженый корабль недоступными для выстрела клетками.
Игра продолжается до уничтожения всех кораблей одного из играющих.

3. Разработка

3.1 Viewer (оболочка)
На форме 2 поля 10х10 клеток. Ниже полей показаны корабли. Свои корабли перед началом игры выставляем на игровое поле. Корабли противника подсвечиваются по мере уничтожения. Над полями надпись о принадлежности хода.
Клетки поля противника могут быть в 3х состояниях: свободное поле, доступное для выстрела; поле попадания; неактивное, окружающее уничтоженый корабль.
Правее находятся 3 кнопки - "НАчать игру" и  "Сдаться", "Расставить корабли автоматически"

3.2 Модель
Перечисление - Мимо, Ранил, Убил, Конец боя.
Перечисление статуса координат - пусто, корабль, выстрел, попадание.
Перечисление типы кораблей : количество палуб
Перечисление направления кораблей : горизонтально, вертикально
Поле в котором храняться координаты своих кораблей - двумерный массив, элементами которого являются: 0 - пустое поле, 1- есть корабль или его часть, 2 - выстрел, 3 - попавший выстрел.
Поле в котором храняться координаты кораблей противника - двумерный массив, элементами которого являются: 0 - пустое поле, 1- есть корабль или его часть, 2 - выстрел, 3 - попавший выстрел.
Поле количества неубитых частей кораблей. При инициализации - 20.
Поле статуса последнего выстрела.
Поле статуса последнего попадания.
Поле ранения логическое.
Поле координат последнего выстрела(строка из 2 цифр).
Метод автоматического заполнения координат кораблей.
Метод заполнения координат кораблей вручную.
Метод выстрела компьютером(генерация случайных координат).
Метод выстрела игрока

3.2.1 Метод выстрела игрока
Входящие параметры - координаты выстрела(х,у).
Результат - перечисление статуса выстрела.
При пустом поле координат возарвщаем "Мимо".
При не пустом поле координат проверяются точки с координатами х+1,х-1,у+1,у-1 для анализа статуса корабля.  Если в этих координатах есть "корабль", то возарвщаем "ранил", иначе "убил". Меняется статус координаты в поле координат.
Уменьшаем счетчик неубитых частей кораблей. Если он стал равен 0, то возвращаем "Конец боя". 

3.2.2 Метод выстрела(генерация случайных координат).
Входящих параметров нет. Возвращает строку из двух цифр - координат массива кораблей противника. 
Проверяем статус последнего выстрела. Если статус "Убил", то поле ранения ложь. Если статус "Мимо" или "Убил" и поле ранения ложь, то генерируем случайное число.

Если ранил , то в масиве координат противника проверяем координаты х-1,х+1,у-1,у+1. Если есть выстрелы по этим координатам, то генерируем координаты в противоположную сторону.
Если выстрелов не было (первое попадание), то случайным образом выбираем одно из проверяемых значений.
Если статус "Мимо" и поле ранения истина, то проверяем координаты х-2,у-2,х+2,у+2.
При генерации проверяем полученные координаты, если в массиве элемент с этими координатами не пустое поле, то генерируем новые координаты.+

3.2.3 Метод заполнения координат кораблей вручную.
Принимает координаты начала корабля, тип корабля, направление размещения. 
Направление арзмещения логическое, по умолчанию ложь - вниз.
Заполняет массив координат кораблей.

3.2.4 Метод удаления координат корабля.
Принимает координаты начала корабля, тип корабля, направление размещения. 
Направление арзмещения логическое, по умолчанию ложь - вниз.
Удаляет из массива координат корабль.

3.2.5 Метод очистки координат кораблей.
Входных параметров нет. Очищает массив координат кораблей.
